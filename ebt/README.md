# EBT — Energy-Backed Token

> **1 EBT = 1 kWh. Always. Immutably. Forever.**

EBT is the unit of account for the PollPower energy economy. One token represents one verified kilowatt-hour of renewable energy generated by a registered producer.

## EBT v5.2 (production)

**Address (Midnight Preview):** `4120b44ed9067f5576006a559e187a447c667db401d9c2ef1d44dedb34e3f835`
**Deployed:** 2026-06-12
**Audit:** [2026-06-10 Fable 5 review](../README.md#audit-2026-06-10)

### What v5.2 fixes (audit findings)

- **C-1:** Producer bound into 4-field HAT signed payload — prevents frontrunning of permissionless `settle()`
- **M-1:** `attestationKey == hash(producer, meterKeyHash)` enforced in-circuit — prevents arbitrary attestation pairing
- **M-4:** `manualReissue` now capped (1M base units), rate-limited (1hr cooldown), and requires Meter Authority co-signature
- **L-1:** `transferOwnership` guards `.is_left` — can't brick the contract by transferring to a ContractAddress
- **L-3:** Audit timestamps populated from caller-supplied time, validated against block time via `blockTimeGte`
- **I-1:** Header correctly says "EBT Token v5.2"

### How settle() works in v5.2

```
sessionPayloadHash = persistentHash([sessionID, hatPubkey, amount, producer])
                                                                    ^^^^^^^^
                                                              C-1: bound into signature
```

The HAT (gateway hardware) signs all four fields. A valid settlement requires:
1. HAT signature over the 4-field payload ✓
2. Meter Authority attestation over the HAT pubkey ✓
3. `attestationKey` matches `hash(producer, meterKeyHash)` ✓
4. Slice amounts satisfy the BPS margin policy ✓
5. Session ID not in `_settledSessions` (replay guard) ✓
6. Authority attestation not expired (blockTimeLte freshness) ✓
